## 项目简介
在跨语言会议、线上课程、涉外交流等场景中，普遍存在**音频杂音、回声干扰、翻译延迟、语种适配不全、采集断流**等问题。本项目采用**Electron 桌面客户端 + Python 中转服务 + 云端 AI 能力**的三层架构，基于 WebSocket 长连接实现全链路流式数据传输，支持多路音频混合采集、专业音频降噪优化、六十余种语言实时同声传译。

应用使用 WebRTC 标准 3A 音频算法优化音频质量，搭配自定义热词词典提升专业场景翻译准确率，同时完成 Electron 安全加固，杜绝密钥泄露风险，是一套功能完整、稳定性强、的桌面同声传译工具。

## ✨ 核心功能
### 1. 多路音频混合采集
支持**电脑系统立体声**与**外部矩阵麦克风**双音源独立/混合采集，兼容会议矩阵设备、线上直播、线下混录等复杂收音场景，音源可实时切换。

### 2. WebRTC 3A 音频全优化
原生集成行业标准音频处理算法，无需额外部署模型，边缘设备亦可流畅运行：
- **AEC 回声消除**：阻断 AI 播报声音被二次采集形成音频环路；
- **ANS 噪音抑制**：自动过滤环境杂音、电流声、背景干扰音；
- **AGC 自动增益**：智能均衡人声音量，避免音量忽大忽小。

### 3. 实时流式同声传译
对接阿里云通义千问实时接口，音频分片流式上传，实现**边采集、边识别、边翻译**。页面分层展示**待确认草稿文本**与**已确认正式文本**，跟随人声语速，保证会话内容。

### 4. 多语种智能适配
原生支持 **60+ 全球语种**双向互译；针对无语音合成（TTS）能力的语种，系统自动禁用 AI 播报按钮，并弹出友好提示，规避功能异常。

### 5. AI 语音回放
翻译段落结束后，生成标准 24kHz PC 音频流，支持一键回放整段翻译语音，方便内容复盘、二次收听。

### 6. 自定义热词词典
内置可视化热词管理面板，支持批量添加、编辑、删除行业术语、固定短语、专属词汇，热词全局生效，大幅提升垂直领域翻译准确度。

### 7. 稳定启停控制
支持随时**暂停/恢复**音频采集，修复旧版本暂停后流中断、异常重启的 Bug；恢复后自动接续上下文，长时间连续运行稳定可靠。

### 8. 沉浸式桌面体验
基于 Electron 实现无边框自定义窗口，支持窗口最小化/最大化/关闭、字幕视图切换、字体缩放、笔记辅助等配套功能，界面简洁专注。

## 🛠️ 开发实现内容
本项目为独立全栈开发，覆盖**前端桌面、后端服务、音频处理、接口对接、安全加固**全流程，主要完成工作如下：
1. **音频采集模块开发**
   基于 Electron `desktopCapturer` 实现系统内录采集，结合原生 MediaDevices 实现麦克风采集，完成双音源混合逻辑调试，解决多路音源兼容问题。
2. **WebRTC 音频算法集成**
   在web音频采集端开启 AEC/ANS/AGC 三大算法，编写配置逻辑，降低回声、噪音、音量不均衡的问题。
3. **流式数据逻辑开发**
   基于 WebSocket 实现音频帧、文本分片的流式收发，对接云端 `delta`/`done` 标记，完成音频拼接、文本分段、内存缓存清理逻辑。
4. **Bug 修复与稳定性优化**
   定位并修复音频**暂停/恢复时流中断、异常重启**问题，重构音频流状态管理逻辑，提升长时间运行稳定性。
5. **多语种兼容开发**
   编写语种能力校验逻辑，区分支持/不支持语音合成的语种，实现播报按钮自动禁用 + 前端提示交互。
6. **热词管理功能开发**
   实现热词弹窗、增删改查交互，完成热词数据序列化，对接云端翻译接口实现热词全局生效。
7. **Electron 桌面开发**
   完成无边框窗口、自定义标题栏、窗口控制（最小/最大/关闭）、IPC 通信、页面布局、样式适配等前端功能。
8. **后端服务开发**
   使用 FastAPI + WebSocket 搭建中转服务，封装云端 AI 接口请求，统一协议格式，实现密钥隔离与流量转发。

## 🏗️ 系统架构
整体采用 **三层分布式架构**，全链路基于 WebSocket 长连接通信，保证低延迟流式传输：
1. **桌面客户端层（Electron）**
   负责窗口管理、人机交互、多路音频采集、PCM 音频处理、字幕渲染、前端逻辑控制。
2. **本地中转服务层（Python）**
   基于 FastAPI 搭建 WebSocket 服务，作为前后端中转节点，转发音频流与文本流，隔离前端与云端接口，保护密钥安全。
3. **云端 AI 服务层（阿里云通义千问）**
   提供实时语音识别（ASR）、多语言同声传译、AI 语音合成（TTS）三大核心 AI 能力，输出流式文本与 PCM 音频流。

**数据流转流程**
音频采集 → Web Audio API 重采样 → WebSocket 分片上传 → Python 中转 → 云端 AI 处理 → 结果回传 → 前端字幕/音频播放

## 🚀 快速部署 & 运行
### 步骤 1：克隆项目到本地
```bash
git clone https://github.com/torbys/DDTrans.git
cd DDTrans
```

### 步骤 2：配置环境变量
在项目**根目录**新建 `.env` 文件，写入以下配置（替换为自己的密钥）：
```env
# 阿里云通义千问接口密钥（必填）
DASHSCOPE_API_KEY=你的DashScope_API_Key
# 后端服务监听端口（默认 1145，可自定义修改）
SERVER_PORT=1145
```
> 提示：`.env` 文件包含敏感密钥，**严禁提交到公共代码仓库**。

### 步骤 3：安装 Python 后端依赖并启动服务
```bash
# 进入后端目录
cd server
# 安装所有 Python 依赖
pip install -r requirements.txt
# 启动后端服务
cd src
python main.py
```
服务启动成功后，终端会打印监听地址，**保持该终端后台运行**。

### 步骤 4：安装前端依赖并启动 Electron 桌面应用
新开终端，回到项目根目录执行：
```bash
# 安装 Node 依赖
npm install
# 启动桌面客户端
npm start
```
启动完成后，会自动弹出应用窗口。

## 📖 基础使用教程
1. **启动准备**：确保 Python 后端服务正常运行，再打开桌面应用；
2. **参数选择**：在主界面选择**源语言、目标语言**，并勾选音频输入源（麦克风 / 系统音 / 混合音源）；
3. **开始同传**：点击「开始同传」，程序自动采集音频并实时翻译，字幕分层展示；
4. **暂停/恢复**：中途可点击暂停按钮停止采集，恢复后自动接续上下文；
5. **语音回放**：暂停状态下，点击翻译区左侧音频按钮，回放 AI 翻译语音；
6. **热词配置**：点击热词按钮，添加行业术语，提升翻译准确率；
7. **结束使用**：点击停止按钮，清空当前会话，返回初始页面。

## ❓ 常见问题 & 排错
1. **播放 AI 语音出现回声**
   项目已默认开启 WebRTC AEC 回声消除；外放场景建议搭配耳机使用，体验更佳。
2. **部分语种语音按钮置灰无法点击**
   该语种暂无 AI 语音合成能力，属于正常限制，以页面提示为准。
3. **端口 1145 被占用**
   修改根目录 `.env` 文件中的 `SERVER_PORT` 字段，同步前后端端口即可。
4. **图标显示空白**
   境外 CDN 无法访问导致，可将 Remix Icon 替换为本地 SVG/字体图标。
5. **音频暂停后出现断流**
   该 Bug 已修复，若仍出现，请检查 Python 后端 WebSocket 连接状态，重启后端服务。
6. **无法获取音频权限**
   检查系统麦克风、录音权限，给予应用录音、系统音频访问权限。
